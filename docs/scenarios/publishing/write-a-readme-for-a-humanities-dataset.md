## What you are trying to do

You need a human-readable landing page for data that someone else might reuse.

!!! quote "One-sentence version"
    Start with a tiny sample, keep provenance, and only scale up once the workflow survives testing.

## You need

- Dataset files
- Basic project information

## Tools

- [Markdown](https://www.markdownguide.org)

## Workflow

1. Explain what the dataset contains in plain language.
2. Describe file formats and column meanings.
3. State who created it and how to cite it.
4. List known problems and missing data.
5. State license, access, and reuse restrictions.

## Output

`README.md` in the dataset folder.

## Check yourself

- Can a stranger open the data without emailing you?
- Are limitations honest?
- Is citation information visible?

## Common traps

- Writing only for yourself.
- Hiding uncertainty because it feels embarrassing.
- Omitting rights information.

## Practice task

Users exchange datasets with no verbal explanation; if the partner gets lost, the README failed.

## Useful extension

Turn the workflow into a small reusable checklist: input, command or tool action, output, quality check, and known failure mode.
