# Sprint formats

## 30-minute FAQ jam

Good for tired rooms and end-of-semester survival.

1. Pick a question.
2. Draft only the sections: what, inputs, workflow, output, traps.
3. Add one tiny example.
4. Trade with another pair for a sanity check.

Deliverable: one rough answer page.

## 60-minute tool surgery

Good when you want students to learn that tools are not magic.

1. Choose a tool and a tiny input.
2. Run the tool.
3. Screenshot or record the key steps.
4. Write down every confusing moment.
5. Convert confusion into “Common traps”.

Deliverable: tested workflow plus traps.

## 90-minute source-to-workflow lab

Good for archival, corpus, or data-modelling classes.

1. Start from one source: PDF, table, interview, letter, website, map, or dataset.
2. Define what output would be useful.
3. Build the smallest workflow.
4. Add provenance and rights notes.
5. Peer-test.

Deliverable: answer page plus example output.

## Two-week mini-release

Good as a final assignment.

Week 1: claim question, draft answer, test workflow.  
Week 2: peer review, revise, merge, present.

Deliverable: polished page, peer-review note, five-minute demo.
