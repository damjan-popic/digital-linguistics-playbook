# Classroom use

This repository works best as a semester-long public-good machine: students do small useful things, test each other’s work, and leave the next cohort a better starting point.

## Three easy ways to use it

<div class="grid cards" markdown>

-   **FAQ sprint**

    Students claim one question, write one page, and test another group’s page.

    [:octicons-arrow-right-24: Sprint formats](sprints.md)

-   **Tool autopsy**

    Students try a tool, document where it breaks, and turn that into “common traps”.

    [:octicons-arrow-right-24: Rubric](rubric.md)

-   **Source-to-answer lab**

    Students start with a messy source and produce a workflow others can reuse.

    [:octicons-arrow-right-24: Question bank](question-bank.md)

</div>

## The golden rule

A classroom contribution should be **tested by someone who did not write it**. If another group cannot reproduce the result, the answer is not done yet. That is not failure; that is the whole point.
