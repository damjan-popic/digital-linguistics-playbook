# How do I import a termbase into memoQ?

<div class="answer-meta" markdown>
<span>CAT tools</span>
<span>beginner</span>
<span>10–25 min</span>
</div>

## What you are trying to do

You want term recognition and terminology QA inside a memoQ project.

!!! quote "One-sentence version"
    Start with a tiny sample, keep provenance, and only scale up once the workflow survives testing.

## You need

- TBX, CSV, or memoQ-compatible termbase file
- memoQ project with language pair

## Tools

- [memoQ](https://www.memoq.com)

## Workflow

1. Open **Resources → Term bases**.
2. Create or select a termbase.
3. Choose **Import** and map languages.
4. Set forbidden or deprecated terms where needed.
5. Attach the termbase to the translation project.

## Output

Active termbase with live recognition in the translation grid.

## Check yourself

- Do terms appear while translating?
- Are source and target languages correct?
- Does QA flag forbidden terms?

## Common traps

- Importing into the wrong language pair.
- Missing term statuses.
- Leaving duplicate concepts unresolved.

## Classroom version

Let students translate five sentences twice: once without the termbase, once with it. Compare speed and consistency.

## Useful extension

Turn the workflow into a small reusable checklist: input, command or tool action, output, quality check, and known failure mode.
